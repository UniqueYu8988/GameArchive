# GameArchive 项目知识复盘 (KNOWLEDGE_BASE)

恭喜！你现在拥有了一个属于自己的现代化游戏馆藏艺廊。通过这个项目，我们不仅实现了一个精美的网页，还涉及到了一些非常实用的编程概念。

## 🎓 核心知识点通俗总结

### 1. 静态网站生成 (Static Site Generation, SSG)

现在的架构已经从“动态餐厅”升级为**“冷冻便当” (Pre-rendered)**。

- **之前 (动态)**：你每次打开网页，后端都要现场去硬盘找图片，像现做菜一样。
- **现在 (静态)**：我们用 `static_export.py` 提前把所有图片和数据打包好（装进便当盒）。用户访问时，直接拿到做好的成品，速度极快且不需要你开电脑。

### 2. Netlify Drop (边缘节点托管)

- 我们把打包好的 `dist` 文件夹交给了 Netlify。
- 它就像一个全球连锁的自动化便利店，把你的“便当”（网站文件）复制到了全世界的服务器节点上。
- **结果**：无论朋友在哪，都能就近获取内容，而且不需要你维护服务器。

### 3. API 与 JSON

- 虽然没有了运行中的 Python 后端，但我们生成了一个 `static_data.json` 文件。
- 这就像一张“固定的菜单”，前端网页直接读取这就知道有哪些游戏了。

### 4. JSON (数据交换格式)

这是一种轻量级的数据格式，像是一种结构化的文本。

- 后端把游戏名字、年份和图片路径打包成 JSON 格式发送，前端能非常轻松地理解并渲染它们。

### 4. 实时监测与轮询 (Polling)

- 我们在网页中设置了一个“定时器”，每隔 10 秒去问一次后端：“有没有新照片？”。这样你只要往文件夹放图，网页很快就能发现并显示出来。

## 🚀 深入学习建议

如果你想进一步探索编程，以下是两条建议：

1.  **想改变网页样子？** 学习一下 **CSS Grid 和 Flexbox**。这是网页布局的灵魂，掌握了它们，你就能随心所欲地控制图片的大小和排列方式。
2.  **想让朋友在全球访问？** 了解一下 **内网穿透 (Intranet Penetration)**，比如 `Cloudflared` 或 `Ngrok`。它们可以给你的本地电脑生成一个唯一的网址，让任何地方的人都能通过手机看到你的游戏馆藏。
